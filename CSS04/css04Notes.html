CSS Layout Notes

  • Flexbox: a method of displaying elements on a webpage that allows for greater formatting options than those available to block and inline only. For example, flexbox allows three articles to be ordered one next to the other, with each taking up the most of their available space in instead of only the space their content takes up.
  • To make something use the flex display, set its parent element to display: flex;. This will cause each child of that element to display as flex elements.
  • Elements laid out as flexible boxes are laid along two axes: main axis and cross axis.
  • flex-wrap: wrap; use inside flexbox container to allow flex items to wrap below each other.
  • flex-basis: xxpx; use to set how wide a flex item will be before it is moved to a new line. Applied to flex items, not flex containers.
  • flex-grow: xx Used as a unitless value to dicate what proportion of the line each flex item will use. If different items are given unique values, they will take up more/less of the line as appropriate. This can also have another unit given, as before: flex: 1 200px; Here, each flex item will have a maximum of 200px, then any remaining space will be divided between each flex item according to their proportions.
  • The flex: shorthand sets, in order: flex-grow, flex-shrink, flex-basis.
  • flex-flow: row wrap shorthand for flex-wrap and flex-direction: (row/column(default)).
  • aling-items: (stretch/center/start/end/flex-start/flex-end/more-available); controls where flex items align on the cross axis.Default value is stretch.
  • justify-content: (start/center/space-around/space-between/space-evenly) determines where the flex items lay on the main axis. Default value is flex-start.
  • order: x sets the position of a flex item. By default, all flex items have an order value of 0. Those with a higher value appear later. Negative values will cause the item to appear earlier. Flex items with the same order value will appear in their source order.
  • Grids allow us to organize our page into columns, rows, and the spaces between (normally called gutters), and keep this layout consistent among pages with different content. We use display: grid; to set our layout to grid, just like setting our layout to flex.
  • Set the number of columns used grid-template-columns: xxpx xxpx xxpx ..., using px or a percentage, and declaring the width for as many columns as you need. Also available is the fr unit, which gives each column a portion of the width equal to the part of the whole the value it is given is. For example: grid-template-columns: 1fr 1fr 2fr; would give each of the first two columns one quarter of the total width (1/4) and the last column half the total width (2/4, 1/2). The number of columns can be repeated using grid-template-columns: repeat(x, y); where x is the number of times that y should be repeated. So grid-template-columns: repeat(2, 1fr); would repeat a listing twice with 1fr of the width of the row before moving to the next row. Similarly, grid-template-columns: repeat(3, 200px) would have three listings per row at 200px wide each, before moving to the next row.

  • Gaps for spaces between rows and columns we use grid-column-gap, grid-row-gap, or grid-gap. Gap values can be lengths or percentages, but not fr. While grid-gap is still supported, gap will be the new standard, and so both should be used to maintain the code for both old and new browsers.
  • Implicit and Explicit Grids Implicit grids occur when we have not declared anything for one axis of grid-template-(columns/rows). Explicit grids are for when we have declared a value for either axis of the same. Implicit grids automatically adjust their width or height to fit the content, but can be given values with grid-auto-(rows/columns). This will ensure that the columns or rows given that value are always that value.
  • grid-auto-(rows/columns): minmax(x,y); IMPORTANT! here, and below! This will allow the axis to which it is set to have a minimum size value of x and a maximum size value of y, based on the content inside the content box (see the first box in the example grid above). This would have solved the issue of content flowing out of the buttons on the Knave character generator.
  • grid-template-(columns/rows): repeat(auto-fill,minmax(x, 1fr)); This sets the size of the axis to a minimum width of x, and will expand as needed to fill the available space. This is different from using only minmax because it will add rows/colums as needed to make space for the contant to fit, not just adjust the width/height of the box it is in.

  • Grid Lines Grids start with lines. In languages like English, lines are numbered from left to right starting with 1. In languages like Arabic, lines are ordered right to left starting with 1. We can place content on our grid using these properties: grid-column-start, grid-column-end, grid-row-start, grid-row-end, grid-column, grid-row, where <>grid-column and grid-row are shorthand for grid-(column/row)-start, grid-(column/row)-end. grid-(row/column)'s values are expressed by separating them with a forward slash like so: grid-column: 1/2;.
